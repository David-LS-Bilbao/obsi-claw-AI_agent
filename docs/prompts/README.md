# README.md

## Propósito

Definir una norma mínima para prompts de dirección y prompts operativos de Codex dentro del proyecto.

## Prompt de director

Debe incluir:

- objetivo del tramo o sprint;
- contexto confirmado;
- límites de seguridad;
- criterio de done;
- y formato de salida esperado.

## Prompt de Codex

Debe incluir:

- rol operativo esperado;
- rutas y repos de referencia;
- restricciones explícitas;
- regla de no inventar estado;
- y obligación de marcar supuestos y pendientes de verificación.

## Regla práctica

Si un prompt puede inducir cambios sobre producción o sobre el runtime real, debe pedir primero auditoría y evidencia antes de ejecución.
